  - `名称`

### 核算工作台

#### 业务全局监控
- 页面交互整体要求：图文并行，整体展示各个指标或字段
- 展示内容：
  - `会计期间`：取当前会计期间
  - `本期应核算项目`：
    - 按照项目类型（整装、软装、设计费）分别展示，区分在建、竣工项目，并显示数量和占比
    - 取数逻辑：
      1. 核算方法=完工百分比法，本期应核算项目满足以下条件：
         - 开工日期不为空且<=当期会计期结束日期
         - 合同状态=打开
         - 项目类型=整装
      2. 核算方法=完成合同法，本期应核算项目满足以下条件：
         - 财务确认竣工日期不为空且<=当期会计期结束日期
         - 合同状态=打开
         - 项目类型=软装
      3. 核算方法=收款确收法，本期应核算线项目满足以下条件：
         - 合同状态=打开
         - 项目类型=设计费
         - 收款净额 ≠ 0
  - `核算进度`：
    - 根据本期应核算项目，按照项目类型（整装、软装、设计费）分别展示各类型项目的核算完成、核算中数量、未开始的数量及占比
  - `项目检查通过率`：
    - 根据本期应核算项目，按照项目类型（整装、软装、设计费）分别展示检查通过数量/同项目类型本期应核算数量
  - `软装竣工出库进度`：
    - 应出库合同总数量：合同状态=打开，项目类型=软装，财务确认竣工日期小于等于当前会计期间结束日期
    - 已出库合同数量：应出库合同有对应竣工销售出库，单据状态=生效
    - 进度：已出库数量/应出库合同总数量
    - 备注：理论上是出库应该和财务确认竣工月份一致且都属于当前会计期间，但实际存在对账不平导致延期核算的情、或者合同履约成本晚于确认竣工月份，导致三者不都等于当前会计期间
  - `软装出库成本获取进度`：
    - 已出库合同数量：应出库合同有竣工销售出库，单据状态=生效
    - 已获取成本合同数量：已出库合同数量标识了获取成本金额时间
    - 进度：已获取成本合同数量：已出库合同数量标识了获取成本金额时间/已出库合同数量
  - `软装对账成功率`：
    - 对账成功合同数量：合同状态=打开，项目类型=软装，财务确认竣工日期小于等于当前会计期间结束日期
    - 对账成功合同数量：应出库合同对应的对账单状态=对账成功
    - 成功率：对账成功合同数量/对账成功合同数量
  - `最近动态`：
    - `获取EBS科目`：获取合同数量，获取成功率
    - `获取金蝶软装工费`：更新合同数量，更新成功率
    - `软装三方对账`：对账数量，对账成功率
    - `软装竣工出库`：出库合同数量，自动出库成功率

- 打开方式：点击 `新增` 按钮
- 表单字段：
  - `财务组织`：下拉多选，支持全选
  - `核算期间`：只能核算当前会计期间，锁定，自动带出
  - `合同号`：非必填，输入后下拉多选，不填默认没有合同过滤条件
  - `是否重算已计算的合同`：布尔，默认不勾选
    - 勾选，则合同对应的软装销售收入计算单范围条件如下：（原则：增量首算和存量反复）
      1.（计算状态=未计算or计算失败or已计算）or 还未创建软装销售收入计算单
      2.项目核算确认状态=未确认 且 没有审批中的项目收入确认审批流程
    - 未勾选，则需要满足条件：
      - （计算状态=未计算 or 计算失败） or 还未创建软装销售收入计算单
  - `是否重算已确认的合同`：
    - 勾选，则合同对应的软装销售收入计算单范围条件：项目核算确认状态=已确认
- 按钮：
  - `确认`：
    - 交互逻辑：
      1. 点击确认后，创建一个计算任务，确认核算范围：在本期应计算项目条件基础上加上根据新增任务选择的项目类型、核算方法、合同号、是否重算（计算状态、确认状态），但需排除掉合同有计算批次且对应批次计算任务正在计算中
      2. 确认了项目核算范围后，创建或更新项目核算单据：
         - 2.1 增量单据：
           - 2.1.1 未创建销售收入计算单的合同，则创建软装销售收入单据，单据类型=竣工收入分摊，是否重算=否，并更新当前计算批次号
           - 2.1.2 重算已确认的合同（其软装销售收入单据的项目核算状态=已确认的合同），则创建项目核算单据，单据类型=重算收入分摊，是否重算=是，并更新当前计算批次号
         - 2.2 存量单据：
           - 2.2.1 计算状态=已计算 且 项目核算确认状态=未确认 且 项目核算无收入确认审批流的合同，则直接更新计算批次号
           - 2.2.2 计算状态=计算失败 or 未计算，则直接更新计算批次号
      3. 根据核算范围进行计算，页面通过进度条展示进度
      4. 退出界面则自动后台计算
      5. 计算后弹框提示“你有软装收入计算任务已经计算完成，请查看详情！”点击后跳转查询核算报告
  - `取消`

#### 软装销售收入计算逻辑
- 步骤1：确认核算范围，若为重算已确认合同，则新增软装收入计算单，否则更新已有软装销售收入计算单
  - 单据编号：新增，自动生成，编码规则：JSD+yymmdd+8位
  - 单据类型：
    - 重算已确认合同：`重算收入分摊`
    - 非重算已确认合同：`竣工收入分摊`
  - 对账单号：取该合同对应的对账单号
  - 会计期间：取当前会计期间
  - 财务主体：取项目基础资料
  - 合同号：取项目基础资料
  - 财务确认竣工日期：取项目基础资料
  - 待分摊金额：取项目基础资料应收总额
  - 税率：取项目基础资料
  - 计算批号：创建计算任务时反写计算批号
  - 是否重算：若为重算合同，则为是，否则为否
  - 重算单号：若为重算合同，则取被重算的单据编号
  - 出库单号：取销售出库单头的单据编号
  - 出库日期：取销售出库单头的业务日期
  - 计算异常信息：计算任务结束后根据计算报告反写异常信息
- 步骤2：采集信息
  - 非重算已确认合同：
    - 采集该合同的销售出库单信息作为销售收入计算分摊明细，满足条件：单据条件=生效，单据类型=软定销售出库
    - 字段：
      - `序号`：取销售出库单分录的行号
      - `单据类型`：取销售出库单头的单据类型
      - `物料编码`：取销售出库单分录的物料编码
      - `物料名称`：取销售出库单分录的物料名称
      - `批号`：取销售出库单分录的批号
      - `库存数量`：取销售出库单分录的出库数量
      - `库存单位`：取销售出库单分录的出库单位
      - `税率`：取项目基础资料税率
      - `客户价`：取销售出库单分录的客户价
      - `项目经理价`：取销售出库单分录的项目经理价
      - `来源行号`：取销售出库行号
  - 重算已确认合同：
    - 仅采集项目的应收总额，其余信息与被重算单据一致
- 步骤3：计算前异常检查合同
  - 非重算已确认合同，满足以下任一条件则为异常合同：
    1. 被分摊的物料数量均等于0（无可分摊的物料）
    2. 被分摊的物料存在数量小于0的数据（仅提醒，计算时不参与收入分摊即可）
    3. 应收总额<0
    4. 所有物料客户价全部=0
    5. 对账状态不等于对账成功
    6. 该合同已生成竣工销售出库单
    - 若存在以上情况，除第二种情况直接更新计算状态=计算失败，将检查点更新在异常信息字段中，第二种情况检查通过继续往下计算，同时将检查点更新在异常信息字段中
    - 重点测试场景：被分摊的物料数量存在大于0的数据，但应收总额=0也支持继续计算，只是收入都为0，也需要出库
  - 重算已确认合同，满足以下任一条件则为异常合同：
    - 重新采集后的应收总额与被重算计算单的应收总额一致
- 步骤4：数量转化
  - 非重算合同才执行，重算合同直接下一步
  - 前提：
    - 物料主单位不等于销售单位时，换算销售单位
    - 物料存在单位转换系数
  - 销售数量=库存数量*物料单位转换系数
- 步骤5：分摊逻辑
  - 计算公式：
    - 价税合计 =销售数量*客户价/∑（销售数量*客户价） × 待分摊金额
    - 税额 = 价税合计 / (1+税率) × 税率
    - 不含税金额 = 价税合计 - 税额
    - 含税单价 = 价税合计 / 销售数量
    - 不含税单价 = 不含税金额 / 销售数量
  - 金额精度：
    - 价税合计、税额、不含税金额：保留2位小数
    - 不含税价格、含税单价：保留6位小数
  - 差异调整规则：
    - 允许差额 = 行数 × 0.005
    - 如实际差额 > 允许差额，报错不允许计算
    - 否则将差额分配到价税合计最大的、且无重复物料的行中（因为重复的有可能退货了）
- 步骤6：计算完成后，生成计算报告，反写销售出库单和软装收入计算单：
  - 若为计算成功，则将软装收入计算单的每行分录的价税合计、含税单价、税额、不含税金额、不含税单价、来源系统、来源单据反写至对账单的来源销售出库单中
  - 若计算失败，则将失败原因更新至软装收入计算单的计算异常信息字段

#### 项目计算逻辑
- 步骤1：核算前检查
  - 整装：
    1. 项目档案满足条件：
       1.1 未完工项目：预计总收入为>0、预计总成本>0
       1.2 已完工项目：
           - 1.2.1 应收总额≦0
           - 1.2.2 整装收款计税已完成当期最新获取
           - 备注：当期最新的标准尚未确定
    2. EBS获取科目：
       - EBS合同履约成本科目每个月都会获取最新的：本期核算时校验完成合同履约成本2026-10开始到截止当前期间科目发生额获取，获取时间根据目前沟通，至少应该晚于1号24:00之后，这个时间可以配置最好
  - 软装：
    1. 软装销售收入计算完成状态校验
       - 1.1 第一次核算，校验是否完成软装销售收入计算（单据类型=竣工收入分摊），校验通过条件：其计算状态=已计算
       - 1.2 若存在第二次及以后的重算，仅校验是否有未完成的重算计算单（单据类型=重算收入分摊），校验通过条件：其计算状态 = 已计算
    2. 软装出库成本是否获取校验
       - 仅第一次核算时校验，软装销售收入计算单（单据类型=竣工收入分摊）对应的来源出库单是否完成成本获取
    3. 金蝶软装工费获取最新数据校验
       - 3.1 非重算情况，校验核算前已完成当期最新的软装工费获取
       - 3.2 重算情况，校验软装工费是否在本期上次核算确认收入时间之后获取过
       - 备注：当期最新的标准尚未确定
  - 设计费：
- 步骤2：核算前检查通过的合同进行下一步计算
  - 整装（完工百分比）：

      - `项目类型`：取项目基础资料
      - `合同号`：取项目基础资料
      - `合同名称`：取项目基础资料
      - `计税方式`：取项目基础资料

      - `客户名称`：取项目基础资料
      - `客户编号`：取项目基础资料

                 - 单据状态：不能为草稿，应该是生效或审批之类的状态
                 - 单据来源业务系统与合同项目基础资料的来源系统一致
               - 参考逻辑：
                 - 目前已有报表可获取业务侧对账数据源，可参考，具体报表路径为：
                   - 积木云装系统：交付系统 >交付经营 >生活家新项目材料开单情况表（saas）
                   - SVI系统：交付系统>领料订单明细表（svi）
                 - 以对应系统报表（积木云装/SVI）的取数逻辑为基础，额外叠加本次对账的单据条件，二者结合后作为最终取数依据
                 - PS：物料数量汇总合计后为0也需要展示返回（数量为0和无记录是两种情况，因为金蝶有合计为0的明细需要区分是无该物料记录还是净数量为0的情况）

                 - `来源系统`：svi/积木云装

             - 取数逻辑：

                 - 单据状态=已审核

                 - 采购退货单：-数量
                 - 直接调拨单：在建工地仓编码固定为"0005"，调入仓为在建仓（编码0005）且调拨方向为普通时，该调拨单数量记为+数量；调出仓为在建仓（编码0005）且调拨方向为退货时，该调拨单数量记为-数量
               - 关键说明：此处不取金蝶即时库存，仅取历史单据行汇总数据，确保与业务单据期间一致
             - 生成库存明细单据，字段说明：

        4. 执行对账规则：
           - 业务和金蝶对账：
             - 匹配维度：项目来源系统 + 项目合同号 + 物料编码 + 收入核算期间
             - 比对内容：同一匹配维度下，对应业务系统（SVI/积木云装）汇总数量 与 金蝶汇总数量 比对（无基准偏向，分别取双方全量数据）
             - 对账结果枚举：待对账/匹配/数量差异
               - 【匹配】：
                 1. 对应业务系统有该物料有效记录、金蝶有该物料有效记录，且双方数量相等（不含业务系统返回的"正负合计为0"场景）
                 2. 双方净数量均为0：业务系统该物料正负合计为0（按现有报表规则，不会返回数据，按照本次对账要求，只要有物料记录，合计为0也需要返回）；金蝶有该物料有效明细（含两条及以上正负抵消单据），且金蝶汇总数量为0，若业务系统违反回该物料库存明细，则占位为0，双方净数量一致，视为无差异，无需修正
               - 【数量差异】：双方均有该物料有效记录，仅数量不相等（含一方数量为0的有效记录，如业务数量10、金蝶数量0，或反之；若业务系统无记录或金蝶无记录默认占位，数量为0）
             - 对账通过条件：该项目归属的业务系统与金蝶的所有物料均为【匹配】状态，无【数量差异】场景
           - 金蝶和EBS对账：
             - 匹配维度：项目来源系统 + 项目合同号 + 物料编码 + 收入核算期间
             - 比对内容：同一匹配维度下，金蝶汇总数量 = EBS数量
             - 对账结果枚举：待对账/匹配/数量差异
               - 【匹配】：
                 1. 金蝶与EBS的数量完全相等
                 2. 双方净数量均为0：金蝶有该物料有效明细（含两条及以上正负抵消单据），且金蝶汇总数量为0，若EBS无该物料库存余额返回，则占位为0，双方净数量一致，视为无差异，无需修正
               - 【数量差异】：双方均有该物料有效记录，仅数量不相等（含一方数量为0的有效记录，如业务数量10、金蝶数量0，或反之；若业务系统无记录或金蝶无记录默认占位，数量为0）
             - 对账通过条件：所有物料均为【匹配】状态，无【数量差异】场景
           - 三方对账成功条件：三方对账成功 = 业务和金蝶对账通过 AND 金蝶和EBS对账通过
        5. 对账完成后回写，三方对账单对账状态及库存明细对账状态
           - 5.1 对账成功时，页面顶部显示"三方对账成功，即将执行出库"提示语
           - 5.2 对账失败时，页面顶部显示"对账失败，请修正数据后重新对账！"提示语，且支持一键复制差异清单（含表格所有差异数据），发送给业务人员
        6. 支持批量操作，逻辑相同
    - `出库`：
      - 交互规则：
        1. 触发前置条件（按钮可点击的前提）：
           - 1.1 对账状态=对账成功（参数：软装出库是否校验对账状态=是，否则不校验）

        2. 校验逻辑，不通过则终止流程：

           - 2.4 一个对账单只能生成一个销售出库单（防重）

                 - `税额`：应收明细行税额之和
                 - `不含税金额`：应收明细行不含税金额之和

                 - `获取成本时间`：
                 - `备注`：
                 - `同步EBS标识`：应收明细行及成本明细行所有同步EBS标识都为是是，值为是，否则为否
               - 明细行信息：
                 - `行号`：锁定，自动生成
                 - `业务类型`：待定
                 - `子类型`：待定
                 - `销售订单号`：根据合同号取其项目基础资料-订单号
                 - `物料编码`：取三方账单对应金蝶库存明细的物料
                 - `物料名称`：取三方账单对应金蝶库存明细的物料
                 - `批号`：取三方账单对应金蝶库存明细的批号
                 - `销售单位`：取三方账单对应金蝶库存明细的物料-销售单位
                 - `销售数量`：出库数量*物料单位转换系数
                 - `币别`：人民币
                 - `客户价`：取三方账单对应金蝶库存明细的客户价
                 - `含税单价`：
                 - `不含税单价`：
                 - `税率`：
                 - `价税合计`：
                 - `税额`：
                 - `不含税金额`：
                 - `源单类型`：三方对账单
                 - `源单编号`：三方对账单号
                 - `出库单位`：取三方账单对应金蝶库存明细的库存单位

                 - `成本金额`：
                 - `成本单价`：
                 - `同步EBS标识`：否
        4. 异常处理：

    - 核心前提：
      1. 参与计算固定字段：税率、数量；数量为空时，默认数量=0；税率为空时，默认税率=0
      2. 恒等式：价税合计=不含税金额+税额
      3. 税额、不含税金额、价税合计三个字段的精度取币别的金额精度、不含税单价和含税单价取币别的单价精度
    - 核心规则：
      - 场景A：修改「不含税单价、含税单价、不含税金额、价税合计」→ 以该字段为唯一基准，自动反算其余所有字段（含税额）
      - 场景B：修改「税额」→ 仅调整不含税金额/价税合计，绝对不反算（不含税、含税）单价，修改税额不倒算单价，优先保单价准确性，税额允许微调适配尾差
    - 5种编辑场景：
      1. 修改【不含税单价】（基准）：不含税金额=不含税单价×数量；税额=不含税金额×税率；价税合计=不含税金额+税额；含税单价=不含税单价×(1+税率)
      2. 修改【含税单价】（基准）：不含税单价=含税单价÷(1+税率)；不含税金额=不含税单价×数量；税额=不含税金额×税率；价税合计=不含税金额+税额
      3. 修改【不含税金额】（基准）：税额=不含税金额×税率；价税合计=不含税金额+税额；不含税单价=不含税金额÷数量；含税单价=不含税单价×(1+税率)
      4. 修改【税额】（重点：不倒算单价）：核心逻辑：锁定单价，仅调整金额类字段；不含税金额=价税合计-税额；不含税单价保持不变；含税单价保持不变；保持【价税合计=不含税金额+税额】强制恒等即可
      5. 修改【价税合计】（基准）：不含税金额=价税合计÷(1+税率)；税额=价税合计-不含税金额；不含税单价=不含税金额÷数量；含税单价=不含税单价×(1+税率)
    - 触发时机（交互逻辑）：
      1. 5个可编辑字段：输入完成、失去焦点时，触发对应场景的计算
      2. 税率/数量修改：保留当前所有字段的最终数值，重新按"不含税单价为基准"重算一遍（避免因数量/税率变化导致单价失真）
      3. 字段修改时：若计算后出现尾差，优先调整税额最后1位小数，保证价税合计恒等
      4. 数量=0时，修改单价不会刷新不含税金额、不会覆盖价税合计
  - `不含税单价`：可编辑，精度取币别单价精度

    - 核心前提：
      1. 参与计算固定字段：税率、数量；数量为空时，默认数量=0；税率为空时，默认税率=0
      2. 恒等式：价税合计=不含税金额+税额
      3. 税额、不含税金额、价税合计三个字段的精度取币别的金额精度、不含税单价和含税单价取币别的单价精度
    - 核心规则：
      - 场景A：修改「不含税单价、含税单价、不含税金额、价税合计」→ 以该字段为唯一基准，自动反算其余所有字段（含税额）
      - 场景B：修改「税额」→ 仅调整不含税金额/价税合计，绝对不反算（不含税、含税）单价，修改税额不倒算单价，优先保单价准确性，税额允许微调适配尾差
    - 5种编辑场景：
      1. 修改【不含税单价】（基准）：不含税金额=不含税单价×数量；税额=不含税金额×税率；价税合计=不含税金额+税额；含税单价=不含税单价×(1+税率)
      2. 修改【含税单价】（基准）：不含税单价=含税单价÷(1+税率)；不含税金额=不含税单价×数量；税额=不含税金额×税率；价税合计=不含税金额+税额
      3. 修改【不含税金额】（基准）：税额=不含税金额×税率；价税合计=不含税金额+税额；不含税单价=不含税金额÷数量；含税单价=不含税单价×(1+税率)
      4. 修改【税额】（重点：不倒算单价）：核心逻辑：锁定单价，仅调整金额类字段；不含税金额=价税合计-税额；不含税单价保持不变；含税单价保持不变；保持【价税合计=不含税金额+税额】强制恒等即可
      5. 修改【价税合计】（基准）：不含税金额=价税合计÷(1+税率)；税额=价税合计-不含税金额；不含税单价=不含税金额÷数量；含税单价=不含税单价×(1+税率)
    - 触发时机（交互逻辑）：
      1. 5个可编辑字段：输入完成、失去焦点时，触发对应场景的计算
      2. 税率/数量修改：保留当前所有字段的最终数值，重新按"不含税单价为基准"重算一遍（避免因数量/税率变化导致单价失真）
      3. 字段修改时：若计算后出现尾差，优先调整税额最后1位小数，保证价税合计恒等
      4. 数量=0时，修改单价不会刷新不含税金额、不会覆盖价税合计

## 【期末处理】产品页面功能描述及交互描述

### 启用管理

#### 列表页
- 列表展示：
  - `财务组织`：自动带出所有财务组织
  - `启用模块`
  - `启用期间`：选择会计期间
  - `是否生效`：是/否
  - `操作`：
    - `启用`：启用后，不可修改启用期间

### 参数管理

#### 说明
- `软装出库是否校验对账状态`：布尔，勾选or不勾选，默认勾选
- `强制校验当期应核算项目是否全部核算完成`：布尔，勾选or不勾选，默认勾选

### 库存关账

#### 列表页
- 列表展示：
  - `财务组织`：自动带出所有有启用期间且生效的财务组织
  - `启用期间`
  - `当前期间`：首次启用时，当前期间=启用期间
- 批量操作：
  - `关账`：点击关账后，当前期间顺延至下一会计期间；校验：若无可用会计期间，提示"当前最大会计期间位yyyymm，无法关账"
  - `反关账`：库存反关账时会校验，当前期间期末结账是否已结账，如果期末结账已结账，则需要先把期末结账反结账，然后才能反关账库存

### 项目关账

#### 列表页
- 列表展示：
  - `财务组织`：自动带出所有财务组织
  - `启用期间`：若为空，则可手工选择启用期间
  - `当前期间`：启用时，当前期间=启用期间
  - `检查结果`
  - `操作`：
    - `启用`：当启用期间为空时，显示该按钮；校验启用期间不为空
    - `检查结果`
- 批量操作：
  - `关账`：点击关账后，二次确认，未完成的单据是否顺延至下一会计期间；校验：若无可用会计期间，提示"当前最大会计期间位yyyymm，无法关账"
  - `关账检查`：检查当期应核算项目是否全部核算完成，根据参数"强制检查当期应核算项目是否全部核算完成"勾选，则若存在未核算完成的数据不允许关账，否则不校验；检查三方账单是否完结：当前期间的三方对账单据是否出库=否时
  - `反关账`：库存反关账时会校验，当前期间期末结账是否已结账，如果期末结账已结账，则需要先把期末结账反结账，然后才能反关账库存
