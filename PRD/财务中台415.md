          二、核心规则
          计算优先级分两类
          场景 A：修改「不含税单价、含税单价、不含税金额、价税合计」→ 以该字段为唯一基准，自动反算其余所有字段（含税额）

          三、5 种编辑场景

          价税合计 = 不含税金额 + 税额
          含税单价 = 不含税单价 × (1 + 税率)
          场景 2：修改【含税单价】（基准）
          不含税单价 = 含税单价 ÷ (1 + 税率)
          不含税金额 = 不含税单价 × 数量
          税额 = 不含税金额 × 税率

          场景 3：修改【不含税金额】（基准）
          税额 = 不含税金额 × 税率
          价税合计 = 不含税金额 + 税额
          不含税单价 = 不含税金额 ÷ 数量
          含税单价 = 不含税单价 × (1 + 税率)
          场景 4：修改【税额】（重点：不倒算单价）

          不含税金额 = 价税合计 - 税额
          不含税单价 保持不变
          含税单价 保持不变
          保持【价税合计 = 不含税金额 + 税额】强制恒等即可
          场景 5：修改【价税合计】（基准）
          不含税金额 = 价税合计 ÷（1 + 税率）
          税额 = 价税合计 - 不含税金额
          不含税单价 = 不含税金额 ÷ 数量
          含税单价 = 不含税单价 ×（1 + 税率）

          2、税率 / 数量修改：保留当前所有字段的最终数值，重新按 “不含税单价为基准” 重算一遍（避免因数量 / 税率变化导致单价失真）
          3、字段修改时：若计算后出现尾差，优先调整税额最后 1 位小数，保证价税合计恒等
          4、数量 = 0 时，修改单价不会刷新不含税金额、不会覆盖价税合计

      - 含税单价
        - 可编辑，精度取币别单价精度
        - 一、核心前提
          1.参与计算固定字段：税率、数量
          数量为空时，默认 数量 = 0
          税率为空时，默认 税率 = 0
          2.恒等式：价税合计 = 不含税金额 + 税额
          3.税额、不含税金额、价税合计三个字段的精度取币别的金额精度、不含税单价和含税单价取币别的单价精度
          二、核心规则
          计算优先级分两类
          场景 A：修改「不含税单价、含税单价、不含税金额、价税合计」→ 以该字段为唯一基准，自动反算其余所有字段（含税额）
          场景 B：修改「税额」→ 仅调整不含税金额 / 价税合计，绝对不反算（不含税、含税）单价，修改税额不倒算单价，优先保单价准确性，税额允许微调适配尾差
          三、5 种编辑场景
          场景 1：修改【不含税单价】（基准）
          不含税金额 = 不含税单价 × 数量
          税额 = 不含税金额 × 税率
          价税合计 = 不含税金额 + 税额
          含税单价 = 不含税单价 × (1 + 税率)
          场景 2：修改【含税单价】（基准）
          不含税单价 = 含税单价 ÷ (1 + 税率)
          不含税金额 = 不含税单价 × 数量
          税额 = 不含税金额 × 税率
          价税合计 = 不含税金额 + 税额
          场景 3：修改【不含税金额】（基准）
          税额 = 不含税金额 × 税率
          价税合计 = 不含税金额 + 税额
          不含税单价 = 不含税金额 ÷ 数量
          含税单价 = 不含税单价 × (1 + 税率)
          场景 4：修改【税额】（重点：不倒算单价）
          核心逻辑：锁定单价，仅调整金额类字段
          不含税金额 = 价税合计 - 税额
          不含税单价 保持不变
          含税单价 保持不变
          保持【价税合计 = 不含税金额 + 税额】强制恒等即可
          场景 5：修改【价税合计】（基准）
          不含税金额 = 价税合计 ÷（1 + 税率）
          税额 = 价税合计 - 不含税金额
          不含税单价 = 不含税金额 ÷ 数量
          含税单价 = 不含税单价 ×（1 + 税率）
          四、触发时机（交互逻辑）
          1、5 个可编辑字段：输入完成、失去焦点时，触发对应场景的计算
          2、税率 / 数量修改：保留当前所有字段的最终数值，重新按 “不含税单价为基准” 重算一遍（避免因数量 / 税率变化导致单价失真）
          3、字段修改时：若计算后出现尾差，优先调整税额最后 1 位小数，保证价税合计恒等
          4、数量 = 0 时，修改单价不会刷新不含税金额、不会覆盖价税合计

## 期末处理

### 启用管理

- 列表
  - 财务组织
    - 自动带出所有财务组织
  - 启用模块
  - 启用期间
    - 选择会计期间
  - 是否生效
    - 是/否
  - 操作
    - 启用
      - 启用后，不可修改启用期间

### 参数管理

- 软装出库是否校验对账状态
  - 布尔，勾选or不勾选，默认勾选
- 强制校验当期应核算项目是否全部核算完成
  - 布尔，勾选or不勾选，默认勾选

### 库存关账

- 列表
  - 财务组织
    - 自动带出所有有启用期间且生效的财务组织
  - 启用期间
  - 当前期间
    - 首次启用时，当前期间=启用期间
- 批量操作
  - 关账
    - 点击关账后，当前期间顺延至下一会计期间
    - 校验：若无可用会计期间，提示“当前最大会计期间位yyyymm，无法关账”
  - 反关账
    - 库存反关账时会校验，当前期间期末结账是否已结账,如果期末结账已结账,则需要先把期末结账反结账，然后才能反关账库存。

### 项目关账

- 列表
  - 财务组织
    - 自动带出所有财务组织
  - 启用期间
    - 若为空，则可手工选择启用期间
  - 当前期间
    - 启用时，当前期间=启用期间
  - 检查结果
  - 操作
    - 启用
      - 当启用期间为空时，显示该按钮。
      - 校验启用期间不为空
    - 检查结果
- 批量操作
  - 关账
    - 点击关账后，二次确认，未完成的单据是否顺延至下一会计期间
    - 校验：若无可用会计期间，提示“当前最大会计期间位yyyymm，无法关账”
  - 关账检查
    - 检查当期应核算项目是否全部核算完成，根据参数“强制检查当期应核算项目是否全部核算完成“勾选，则若存在未核算完成的数据不允许关账，否则不校验
    - 检查三方账单是否完结：当前期间的三方对账单据是否出库=否时
  - 反关账
    - 库存反关账时会校验，当前期间期末结账是否已结账,如果期末结账已结账,则需要先把期末结账反结账，然后才能反关账库存。
